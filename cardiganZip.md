# cardiganZip UI 设计规范

## 1. 整体定位

产品定位是 **完全免费、简洁、轻量、可信赖的桌面端压缩/解压工具**。

设计关键词：

**干净、克制、轻量、高级、无广告感、无商业打扰。**

界面不要出现：

- VIP
- 专业版
- 会员
- 升级按钮
- 广告卡片
- 弹窗营销
- 过多功能入口
- 复杂侧边栏

整体感觉应该像系统自带工具的增强版，而不是国产流氓软件。

---

## 2. 主色调

采用 **米白色 + 柔和黄色** 为核心配色。

### 背景色

| 用途    | 色值建议                  |
| ----- | --------------------- |
| 页面主背景 | `#FBF8F1`             |
| 卡片背景  | `#FFFFFF` / `#FFFCF7` |
| 侧边栏背景 | `#FFF9ED`             |
| 分割线   | `#EDE4D5`             |
| 输入框背景 | `#FFFFFF`             |

### 主色黄色

| 用途       | 色值建议      |
| -------- | --------- |
| 主按钮背景    | `#FFC640` |
| 按钮 hover | `#FFB820` |
| 选中态浅黄背景  | `#FFF0C8` |
| 图标点缀黄    | `#F5B21B` |
| 边框黄      | `#F2D28A` |

### 文字色

| 用途   | 色值建议      |
| ---- | --------- |
| 主标题  | `#1F1F1F` |
| 正文   | `#3A3A3A` |
| 次级说明 | `#7A7A7A` |
| 占位文字 | `#B0B0B0` |
| 弱提示  | `#999999` |

整体颜色要 **低饱和、柔和、耐看**，黄色只作为按钮和选中态，不要大面积铺满。

## 3. 字体规范

建议使用系统字体，保证 Windows 和 macOS 都自然。

```css
font-family:
  "Inter",
  "PingFang SC",
  "Microsoft YaHei",
  "Segoe UI",
  sans-serif;
```

### 字号建议

| 场景    | 字号          | 字重        |
| ----- | -----------:| ---------:|
| 页面大标题 | 26px - 30px | 700       |
| 弹窗标题  | 22px - 24px | 700       |
| 卡片标题  | 22px - 26px | 700       |
| 普通正文  | 15px - 16px | 400 / 500 |
| 表格文字  | 14px - 15px | 400       |
| 辅助说明  | 13px - 14px | 400       |
| 按钮文字  | 15px - 16px | 600       |

中文界面不要用太细的字体，标题要稍微粗一点，显得稳定。

## 4. 窗口规范

### 主窗口

主窗口是简洁首页结构。

建议尺寸：

```text
宽度：1200px - 1320px
高度：760px - 860px
圆角：14px - 18px
```

窗口样式：

```css
background: #FBF8F1;
border-radius: 16px;
box-shadow: 0 18px 45px rgba(0, 0, 0, 0.12);
border: 1px solid rgba(120, 100, 70, 0.12);
```

### 单功能窗口

比如“打开压缩包”“压缩”这种脱离主界面的窗口。

建议尺寸：

```text
宽度：980px - 1120px
高度：680px - 760px
圆角：16px
```

特点：

不带主侧边栏，只保留标题栏、内容区、底部操作按钮。

---

## 5. 首页布局规范

首页只包含三个核心板块：

```text
压缩
解压
最近文件
```

不要放其他入口。

### 首页结构

```text
顶部栏
左侧栏
主内容区
  ├── 压缩卡片
  ├── 解压卡片
  └── 最近文件列表
```

### 左侧栏

只保留：

```text
首页
设置
```

首页放上方，设置放底部。

侧边栏宽度建议：

```text
240px - 260px
```

选中状态：

```css
background: #FFF0C8;
border-radius: 12px;
color: #1F1F1F;
```

图标统一使用线性图标，线宽 1.8px - 2px。

---

## 6. 卡片规范

卡片是界面的主要视觉容器。

```css
background: #FFFFFF;
border: 1px solid #EFE5D6;
border-radius: 16px;
box-shadow: 0 8px 24px rgba(80, 60, 30, 0.06);
```

### 卡片间距

```text
卡片内边距：28px - 36px
卡片之间间距：24px
模块上下间距：32px - 40px
```

卡片不要太挤，留白是高级感的关键。

## 7. 按钮规范

### 主按钮

用于：

```text
选择文件压缩
选择压缩包解压
开始压缩
打开文件
开始解压
```

```css
height: 44px;
padding: 0 28px;
border-radius: 10px;
background: #FFC640;
color: #1F1F1F;
font-weight: 600;
border: none;
```

hover：

```css

```

active：

```css
background: #EAA500;
```

### 次按钮

用于：

```text
取消
清空
浏览
预览
```

样式：

```css

padding: 0 24px;
border-radius: 10px;
background: #FFFFFF;
border: 1px solid #E6DDCF;
color: #333333;
```

## 9. 表格规范

用于最近文件、压缩包内容列表。

### 表头

```css
font-size: 14px;
color: #777777;
font-weight: 500;
height: 48px;
```

### 表格行

```css
height: 58px - 64px;
border-bottom: 1px solid #EFEAE2;
```

### 表格内容


```text

```

## 10. 打开压缩包窗口规范

这是一个独立单窗口，不带主侧边栏。

### 布局结构

```text
标题栏：打开压缩包
工具栏：重新选择 / 文件名 / 解压到 / 预览
主体：
  左侧：压缩包目录树
  中间：文件列表
  右侧：压缩包信息 + 预览区域
底部：
  状态统计
  取消 / 打开文件
```

### 功能重点

这个界面主要用于：

```text
查看压缩包内容
选择文件
预览文件
解压到指定位置
打开文件
```

### 信息面板字段

```text
格式：ZIP
压缩大小：12.4 MB
文件数：28
文件夹：6
创建时间：2024-05-20 10:32
```

右侧预览区域不要做太复杂，保持一个浅色虚线框即可。

---

## 11. 压缩窗口规范

也是独立单窗口，不带主侧边栏。

### 布局结构

```text
标题栏：压缩
左侧：
  拖拽上传区域
  已选择文件列表
  总大小
  清空列表
右侧：
  压缩设置
  压缩包名称
  保存位置
  压缩格式
  压缩级别
  可选密码
底部：
  取消 / 开始压缩
```

### 压缩格式选择

使用分段按钮：

```text
ZIP / 7Z / TAR
```

选中 ZIP：

```css
background: #FFF0C8;
border: 1px solid #F2C14E;
color: #1F1F1F;
```

未选中：

```css
background: #FFFFFF;
border: 1px solid #E6DDCF;
color: #333333;
```

## 16. 文案风格

文案要短，不要营销化。

推荐文案：

```text
将文件或文件夹压缩为压缩包
解压压缩包到当前文件夹
拖拽文件或文件夹到此处
选择文件或文件夹
选择压缩包
开始压缩
开始解压
打开文件
打开所在文件夹
```

避免文案：

```text
极速解压
全网最快
专业版
会员专享
立即升级
超强功能
安全卫士
清理加速
```

---

## 17. 设计原则总结

这套 UI 的核心原则是：

```text
1. 功能少，但每个功能清楚。
2. 留白多，不堆功能。
3. 黄色只做强调，不做大面积装饰。
4. 不出现会员、广告、营销入口。
5. 所有窗口都像系统工具一样可信。
6. 主界面负责入口，单窗口负责具体任务。
```

一句话概括就是：

**“像系统自带工具一样简洁，但比系统工具更温暖、更精致。”**
